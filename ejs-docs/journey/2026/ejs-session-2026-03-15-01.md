# Decisions Made
- Decision: Replace `crypto.randomUUID()` with custom `Math.random()`-based UUID v4 generator
  - Reason: Hermes JavaScript engine (used by React Native) does not provide the Web Crypto API's `crypto.randomUUID()` method
  - Alternatives considered: (1) polyfill `crypto` globally, (2) use `expo-crypto` package, (3) use `uuid` npm package, (4) custom Math.random-based generator
  - Impact: All expense and category creation now works on Hermes. UUID uniqueness is sufficient for local SQLite IDs (not security-critical). Low risk, minimal dependency footprint.

- Decision: Place `generateId()` in `db/shared.ts`
  - Reason: Centralized with other DB utility functions; both consumers (`expenses.ts`, `categories.ts`) already import from `shared.ts`
  - Impact: Single source of truth for ID generation across the data layer

# Key Learnings
Technical insights:
- **Hermes does NOT provide `crypto.randomUUID()`** — this is a Web API not available in React Native's Hermes engine. Code that works in Node.js/browser will fail silently (caught by try/catch) or crash on Hermes.
- Detox "app died" / "obituary" messages in logcat are often from `terminateApp()` in `afterAll`, not from a test-time crash. Check `adb logcat -b crash` for actual crashes.
- Espresso's 75% visibility check failing while the view IS found by testID means the view exists but is covered by another view/screen — not that the view is missing.

Prompting insights:
- Console.log debugging + logcat is the most reliable way to trace runtime errors in Detox E2E tests when the UI behavior doesn't match expectations.

Tooling insights:
- `adb logcat -d | grep "TAG"` is essential for debugging React Native runtime errors during Detox tests
- `adb logcat -c` before test runs ensures clean log output

# If Repeating This Work
Do this:
- Check Hermes API compatibility early — `crypto`, `Intl`, and other Web APIs may be missing
- Use `console.log` + logcat as first debugging step for mysterious E2E failures
- Check `adb logcat -b crash` separately from general logcat to distinguish real crashes from normal termination

Avoid this:
- Don't assume `crypto.randomUUID()` works everywhere — it's a Web API
- Don't interpret Detox "app disconnected" as a crash — check the crash buffer first
- Don't try multiple text input approaches when the error is in the save handler, not the input

Watch out for:
- Hermes engine missing Web APIs that work in Node.js/browser/Jest environments
- `try/catch` in save handlers silently swallowing errors and showing Alerts instead of propagating
- Detox afterAll cleanup messages looking like test-time crashes in logcat
