# Key Learnings
- **Parallel specialist dispatch works well** when files don't overlap — 4 agents completed their work without merge conflicts.
- **babel-plugin-dynamic-import-node** is needed for Jest to handle dynamic `import()` in RN projects — static mocking alone isn't sufficient.
- **Debug-signed release APKs** are a pragmatic shortcut for demos but must never ship to production — the debug keystore is publicly known.
- **Detox E2E stability** required explicit waits and careful element selectors; the initial expansion had flaky timing issues.
- **Session store queries** are reliable for recovering context across sessions — the full interaction trail was reconstructable.

# If Repeating This Work
- **Do**: Dispatch accessibility agents in parallel from the start — non-overlapping file ownership makes this safe and fast.
- **Do**: Run the full E2E suite on emulator immediately after expanding scenarios, not after committing — earlier feedback loop.
- **Do**: Generate README early in the project lifecycle, not as a final polish step — it forces clarity on scope and architecture.
- **Avoid**: Don't assume E2E tests will pass on first run after bulk changes — budget time for stabilisation.
- **Watch out for**: Dynamic import mocking in Jest with RN — needs the babel plugin configured in test env only.

# Future Agent Guidance
- **Prefer**: When touching accessibility, audit the entire screen holistically rather than adding props piecemeal — the expense-engineer's approach of doing all a11y for a screen at once was clean.
- **Prefer**: Use `accessibilityRole="progressbar"` with `accessibilityValue` for any progress indicators — the budget-dashboard-engineer set a good pattern.
- **Prefer**: For chart components, generate dynamic summary strings via useMemo — the insights-engineer pattern makes charts meaningfully accessible.
- **Avoid**: Don't create release signing keys in automated sessions — this should be a deliberate human action with proper key management.
- **Avoid**: Don't skip the tsc check between agent merges — type errors from one agent can mask issues in another's work.
