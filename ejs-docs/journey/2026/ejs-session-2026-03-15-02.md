# Decisions Made
1. **useFocusEffect no-op mock**: React's render phase cannot call setState synchronously; using the no-op avoids infinite re-renders while useEffect paths cover the same refresh logic. The useFocusEffect callbacks (2-3 lines per hook) are intentionally uncovered but coverage still exceeds 80%.
2. **Stable array references**: `useMonthlyTrend` deps on `months` array require stable references in tests â€” declared outside `renderHook` callback.
3. **Separate db-index.test.ts**: `db/index.ts` needed its own test file to mock `@/db/migrations` and `@/db/seeds` cleanly without polluting data-layer tests.
4. **Keep E2E suite stable-first**: retain reliable smoke scenarios in the default run while documenting unresolved Budget/Insights flakiness for follow-up.

# Key Learnings
- `useFocusEffect` from react-navigation should always be mocked as a no-op in tests; calling the callback synchronously during render phase causes React setState-during-render violations and infinite re-render loops
- Hook deps that receive new object/array references every render (e.g., inline arrays) will cause useCallback to recreate, triggering useEffect, causing state updates and re-renders â€” pass stable references in tests
- Installing @testing-library/react-native required --legacy-peer-deps due to peer dependency conflicts with react-native version
- Attached-device Detox runs can intermittently disconnect (known emulator instability); retries are sometimes required for clean signal.

# If Repeating This Work
- Always use `jest.fn()` (no-op) for `useFocusEffect` mock
- Declare array/object constants outside `renderHook` callback if passed to hooks that depend on them
- Check `db/migrations/` coverage separately â€” migration `up` functions require integration tests to cover
